# ShopEasy 통합 웹 대시보드

> 2025년 3분기 이커머스 주문 감소 원인을 분석하고,
> 모바일·전자기기 구매 완료율 개선을 위한 A/B 테스트를 제안한 웹 기반 데이터 분석 프로젝트입니다.

- **Web Dashboard**: [https://dffxonnb-cyber.github.io/ShopEasy/](https://dffxonnb-cyber.github.io/ShopEasy/)
- **GitHub Repository**: [https://github.com/dffxonnb-cyber/ShopEasy](https://github.com/dffxonnb-cyber/ShopEasy)
- **Project Type**: 이커머스 주문 감소 원인 분석 / 전환율 분석 / 웹 대시보드
- **Data Type**: 주문 데이터, 사용자 데이터, 세션 데이터
- **Role**: 데이터 설계, 분석, 인사이트 도출, 웹 대시보드 구현, A/B 테스트 설계

## 1. Executive Summary

ShopEasy는 2025년 3분기 동안 전체 주문 수와 완료 주문 수가 함께 감소하는 흐름을 보였습니다.

본 프로젝트는 주문 감소의 원인을 단순 유입 감소로 보지 않고, 
**고객 세그먼트, 상품 카테고리, 진입 페이지, 디바이스, 이탈 페이지** 관점으로 나누어 분석했습니다.

분석 결과, 주문 감소는 다음 요인과 관련이 있는 것으로 나타났습니다.

- 홈·검색 진입 이후 구매로 이어지는 흐름 약화
- 모바일 환경의 낮은 구매 전환율
- 전자기기 카테고리의 낮은 주문 완료율
- 장바구니·결제 단계에서 발생하는 고의도 사용자 이탈

이를 바탕으로 **모바일 환경에서 전자기기 상품의 혜택·배송·반품 정보와 구매 버튼 노출을 개선하는 A/B 테스트**를 제안했습니다.

## 3. Business Problem

ShopEasy는 최근 3개월간 주문 수가 꾸준히 감소하고 있는 상황입니다.

경영진은 단순히 “주문이 줄었다”는 결과가 아니라, 다음 질문에 답할 수 있는 분석 결과를 필요로 했습니다.

| 핵심 질문 | 분석 목적 |
| --- | --- |
| 어떤 유저 세그먼트에서 이탈이 가장 많이 발생했는가? | 고객 이탈 위험군 파악 |
| 어떤 상품 카테고리에서 주문 완료율이 낮은가? | 전환 저하 카테고리 식별 |
| 세션 수 대비 구매 완료율이 낮은 원인은 무엇인가? | 유입·탐색·구매 흐름의 병목 파악 |

단순 랜덤 데이터가 아니라 실제 이커머스 지표 분석 흐름을 연습할 수 있도록 
**주문 감소, 카테고리별 완료율 차이, 디바이스별 전환율 차이, 고객 이탈 신호**가 드러나도록 설계했습니다.

## 5. Analysis Framework

주문 감소 원인을 하나의 숫자로만 판단하지 않고, 다음과 같이 분석 관점을 분리했습니다.

| 분석 관점 | 핵심 지표 | 판단 기준 |
| --- | --- | --- |
| 월별 성과 | 전체 주문 수, 완료 주문 수, 완료율 | 7월~9월 주문 흐름 변화 |
| 고객 세그먼트 | 연령대별 이탈률 | `purchase_count = 0`인 유저 비율 |
| 상품 카테고리 | 카테고리별 주문 완료율 | 전체 주문 중 `order_status = 완료` 비율 |
| 진입 페이지 | 진입 페이지별 구매 전환율 | 세션 중 `purchase_completed = Y` 비율 |
| 디바이스 | 디바이스별 구매 전환율 | mobile / desktop / tablet 전환율 비교 |
| 이탈 페이지 | 이탈 페이지별 세션 수 | 검색, 홈, 장바구니, 결제, 완료 페이지별 이탈 신호 |

분석 흐름은 아래와 같이 구성했습니다.

```text
주문 감소 확인
→ 카테고리별 완료율 비교
→ 진입 페이지별 전환율 확인
→ 디바이스별 전환 차이 확인
→ 고객 이탈 세그먼트 확인
→ 이탈 페이지 분석
→ A/B 테스트 제안
```

## 6. Key Metrics

### 6-1. 월별 주문 수 및 완료 주문 추이

| 월 | 전체 주문 수 | 완료 주문 수 | 구매 완료율 |
| --- | ---: | ---: | ---: |
| 7월 | 420건 | 305건 | 72.62% |
| 8월 | 340건 | 224건 | 65.88% |
| 9월 | 240건 | 159건 | 66.25% |

**해석**

7월 대비 9월 전체 주문 수와 완료 주문 수가 모두 감소했습니다. 전체 주문 수는 420건에서 240건으로 줄었고, 완료 주문 수 역시 305건에서 159건으로 감소했습니다.

이는 단순히 주문 시도만 줄어든 것이 아니라, 실제 구매 완료 규모도 함께 감소했다는 의미입니다. 따라서 주문량 감소뿐 아니라 구매 완료 단계까지 이어지는 전환 흐름을 함께 점검할 필요가 있습니다.

---

### 6-2. 카테고리별 주문 완료율

| 카테고리 | 주문 완료율 | 완료 주문 / 전체 주문 | 평균 객단가 |
| --- | ---: | ---: | ---: |
| 식품 | 81.97% | 150 / 183건 | 69,164원 |
| 뷰티 | 73.63% | 134 / 182건 | 74,879원 |
| 홈리빙 | 72.09% | 124 / 172건 | 162,680원 |
| 패션 | 67.47% | 168 / 249건 | 112,920원 |
| 전자기기 | 52.34% | 112 / 214건 | 270,009원 |

**해석**

전자기기 카테고리의 주문 완료율이 52.34%로 가장 낮았습니다. 동시에 평균 객단가는 270,009원으로 가장 높았습니다.

전자기기는 고가 상품 특성상 가격 부담, 혜택 인지 부족, 배송·반품 조건 확인 과정에서 구매 완료 전 이탈이 발생했을 가능성이 있습니다. 따라서 전자기기 카테고리는 우선 개선 대상입니다.

---

### 6-3. 진입 페이지별 구매 전환율

| 진입 페이지 | 세션 수 | 구매 완료 수 | 구매 전환율 |
| --- | ---: | ---: | ---: |
| 상품상세 | 455회 | 79건 | 17.36% |
| 카테고리 | 319회 | 22건 | 6.90% |
| 홈 | 715회 | 45건 | 6.29% |
| 검색 | 511회 | 28건 | 5.48% |

**해석**

상품상세 진입 유저의 구매 전환율은 17.36%로 가장 높았습니다. 반면 홈과 검색 진입 유저의 구매 전환율은 각각 6.29%, 5.48%로 낮았습니다.

이는 구매 의도가 명확한 상품상세 유입은 실제 구매로 잘 이어지지만, 홈·검색처럼 탐색 단계에서 시작한 세션은 구매까지 이어지는 흐름이 약하다는 의미입니다.

### 6-4. 디바이스별 구매 전환율

| 디바이스 | 세션 수 | 구매 완료 수 | 구매 전환율 |
| --- | ---: | ---: | ---: |
| 태블릿 | 143회 | 24건 | 16.78% |
| 데스크톱 | 508회 | 64건 | 12.60% |
| 모바일 | 1,349회 | 86건 | 6.38% |

**해석**

모바일은 세션 수가 1,349회로 가장 많았지만 구매 전환율은 6.38%로 가장 낮았습니다.

이는 가장 많은 유입이 발생하는 모바일 환경에서 구매 완료까지 이어지는 효율이 낮다는 의미입니다. 모바일 상품상세, 장바구니, 결제 단계의 사용성 개선이 전체 구매 전환율 개선에 중요한 과제가 될 수 있습니다.

50대 이상은 구매 동선의 복잡함, 20대는 가격 민감도나 혜택 부족이 이탈에 영향을 주었을 가능성이 있습니다. 따라서 세그먼트별로 다른 개선 접근이 필요합니다.

## 7. Key Findings

### Finding 1. 주문 감소는 전체 주문과 완료 주문 모두에서 나타났다.

| 월 | 전체 주문 수 | 완료 주문 수 |
| --- | --- | --- |
| 7월 | 420건 | 305건 |
| 8월 | 340건 | 224건 |
| 9월 | 240건 | 159건 |

7월 대비 9월 전체 주문 수와 완료 주문 수가 모두 감소했습니다. 이는 주문 시도뿐 아니라 실제 구매 완료 규모도 줄어든 상황으로 볼 수 있습니다.
단순히 유입을 늘리는 것만으로는 충분하지 않으며, 유입 이후 상품 탐색, 장바구니, 결제 완료까지 이어지는 전환 흐름을 함께 개선해야 합니다.

### 테스트 주제

**모바일·전자기기 구매 완료율 개선 테스트**

전자기기 카테고리는 주문 완료율이 가장 낮고, 모바일은 세션 규모가 가장 크지만 구매 전환율이 낮았습니다.
따라서 모바일 환경에서 전자기기 상품을 탐색하는 사용자를 대상으로 구매 완료율 개선 실험을 설계했습니다.

| 항목 | 내용 |
| --- | --- |
| 가설 | 모바일 전자기기 상품상세/장바구니 화면에서 혜택·배송·반품 정보를 명확히 노출하면 구매 완료율이 개선될 것이다. |
| 대조군 A | 기존 모바일 상품상세 및 장바구니 화면 |
| 실험군 B | 혜택 요약, 배송·반품 안내, 고정 구매 버튼을 강화한 모바일 상품상세 및 장바구니 화면 |
| 주요 KPI | 구매 완료율 |
| 보조 KPI | 장바구니 이탈률, 결제 시작률, 결제 완료율 |
| 테스트 대상 | 모바일에서 전자기기 카테고리 상품상세 또는 장바구니에 진입한 사용자 |
| 테스트 기간 | 최소 2주 |
| 성공 기준 | 실험군 B의 구매 완료율이 대조군 A 대비 상승하고, 장바구니 이탈률이 감소하는 경우 |
| 주의사항 | 할인율 변경 없이 정보 제공 방식과 UI 개선만 테스트해 가격 효과와 UI 효과가 섞이지 않도록 통제 |

## 11. AI Usage

본 프로젝트에서는 생성형 AI를 단순 결과 생성 도구가 아니라, **분석 설계·코드 작성·문서화·대시보드 구현을 보조하는 협업 도구**로 활용했습니다.

다만 AI가 생성한 내용을 그대로 사용하지 않고, 데이터 구조, 지표 정의, 계산 로직, 인사이트 해석, 대시보드 흐름은 직접 검토하고 수정했습니다.

| 활용 단계 | AI 활용 내용 | 직접 검토 및 수정한 부분 |
| --- | --- | --- |
| 문제 정의 | 주문 감소 원인을 분석하기 위한 관점 초안 도출 | 최종 분석 질문을 고객 세그먼트, 상품 카테고리, 세션 전환 흐름 중심으로 재정리 |
| 데이터 생성 | 주문·유저·세션 더미 데이터 구조 설계 | 주문 감소, 전환 차이, 이탈 신호가 드러나는 구조로 수정 |
| 분석 코드 작성 | pandas 기반 지표 계산 코드 초안 생성 | 컬럼명, 값 형식, 집계 기준에 맞게 코드 수정 |
| 오류 해결 | 구매 완료 여부 계산 오류 원인 파악 | `purchase_completed = Y/N` 형식을 반영하도록 로직 직접 수정 |
| 인사이트 도출 | 분석 결과를 문장화하고 리포트 구조 초안 작성 | 수치 근거와 비즈니스 질문이 연결되도록 해석 문장 재작성 |
| 대시보드 구현 | HTML/CSS/JavaScript 대시보드 구성 방향과 UI 문구 제안 | 문제 → 근거 → 해석 → 액션 흐름으로 재구성 |

### Human Review Example

AI 코드 초안에서는 `purchase_completed` 값 중 `true`, `yes`, `1`, `완료`만 구매 완료로 인식했습니다.

하지만 실제 세션 데이터의 구매 완료 값은 `Y/N` 형태였기 때문에, 초기 로직을 그대로 사용할 경우 진입 페이지별 전환율, 디바이스별 전환율, 이탈 페이지 분석에서 구매 완료 수가 모두 0으로 계산되는 문제가 발생했습니다.

이를 해결하기 위해 `Y` 값을 구매 완료로 인식하도록 로직을 수정했습니다.

```python
sessions["is_purchase_completed"] = sessions["purchase_completed"].apply(
    lambda x: 1 if str(x).strip().upper() in ["Y", "YES", "TRUE", "1", "완료"] else 0
)
```

이 과정을 통해 AI가 생성한 코드를 그대로 사용하는 것이 아니라, **실제 데이터 구조와 값 형식에 맞게 검증하고 수정하는 과정**이 필요함을 확인했습니다.

### What AI Did Not Replace

본 프로젝트에서 AI는 분석과 구현을 보조했지만, 다음 판단은 직접 수행했습니다.

- 어떤 비즈니스 질문을 중심으로 분석할지 결정
- 어떤 지표를 핵심 KPI로 볼지 선택
- 계산 결과가 실제 데이터 구조와 맞는지 검증
- 주문 감소 원인을 어떤 흐름으로 해석할지 판단
- 대시보드 섹션 순서와 전달 구조 설계
- 최종 개선안과 A/B 테스트 방향 결정

## 12. Human Review & Improvements

### 12-1. 구매 완료 여부 계산 로직 수정

초기 코드에서는 `purchase_completed` 값 중 `true`, `yes`, `1`, `완료`만 구매 완료로 인식했습니다.

하지만 실제 데이터에는 구매 완료 값이 `Y/N` 형태로 들어가 있었기 때문에, `Y`도 구매 완료로 인식하도록 수정했습니다.

```python
sessions["is_purchase_completed"] = sessions["purchase_completed"].apply(
    lambda x: 1 if str(x).strip().upper() in ["Y", "YES", "TRUE", "1", "완료"] else 0
)
```

이 수정이 없으면 진입 페이지별 전환율, 디바이스별 전환율, 이탈 페이지 분석에서 구매 완료 수가 모두 0으로 계산되는 문제가 발생했습니다.

### 12-2. 분석 흐름 중심 대시보드로 재구성

초기 결과는 분석 결과표와 차트를 나열하는 구조에 가까웠습니다.

하지만 실무 분석 문서에서는 단순 시각화보다 **문제 → 근거 → 해석 → 액션**의 흐름이 중요하다고 판단했습니다. 그래서 대시보드를 주문 감소 확인부터 A/B 테스트 제안까지 이어지는 구조로 재구성했습니다.

### 12-3. 웹 기반 결과물로 확장

초기 과제는 Looker Studio 기반 대시보드 제작을 전제로 했지만, 본 프로젝트에서는 결과물을 더 안정적으로 공개하고 포트폴리오에 연결하기 위해 HTML/CSS/JavaScript 기반 웹 대시보드로 확장했습니다.

- 분석 결과를 포트폴리오 프로젝트처럼 보여줄 수 있음
- 단순 툴 사용을 넘어 분석 결과 전달 방식까지 설계 가능

## 13. Limitations

본 프로젝트는 분석 실습 및 포트폴리오 제작을 위한 더미 데이터를 사용했기 때문에 다음과 같은 한계가 있습니다.

| 한계점 | 설명 | 보완 방향 |
| --- | --- | --- |
| 더미 데이터 기반 | 실제 서비스 로그가 아닌 생성 데이터이므로 실제 고객 행동을 완전히 반영하지는 않음 | 실제 서비스 데이터에서는 이벤트 로그, 사용자 행동 경로, 상품 상세 데이터 추가 필요 |
| 사용자 여정 단순화 | 세션 단위 데이터만으로 실제 퍼널 전체를 완전하게 추적하기 어려움 | view_item, add_to_cart, begin_checkout, purchase 등 이벤트 단위 로그 필요 |
| 이탈 원인 추정 | 전환율과 이탈 페이지를 기반으로 원인을 추정했지만, 직접적인 이탈 사유는 확인할 수 없음 | 설문, VOC, 결제 실패 로그, UI 클릭 로그 등 추가 분석 필요 |
| 통계 검정 미포함 | A/B 테스트 설계안은 제안 단계이며 실제 유의성 검정은 수행하지 않음 | 실제 실험 시 표본 수 산정, 유의수준, 검정력 설정 필요 |
